Prism (棱镜) 使用手册

🌈 Prism (棱镜)：光的折射与思维的绚烂
随手丢进 Memos 的原始笔记，就像一束朴素的白光。当它穿过 Prism 时，被 AI 优雅地折射开来，化作了标签、结构化 Markdown（MinerU 解析）和清晰的知识脉络。零散的想法进去了，出来的是一抹绚丽的知识彩虹。

当前完成状态

已经完成并通过真实测试的范围：

  • Memos、Caddy 网关、Sidecar API、worker 的 Docker Compose 启动。
  • Memos webhook 接入、任务入队、后台处理和管理接口。
  • OpenAI-compatible LLM 整理，默认使用 DeepSeek deepseek-chat。
  • AI 整理 memo 创建，并写入 Memos 原生 REFERENCE relation，由整理 memo 指向原始 memo。
  • 内置 /admin/ui 调试管理页面，可查看任务、重试任务和审核候选标签。
  • 候选标签创建、审核、审核后 active 生效。
  • AI 可从正文自动生成 tag 建议；未审核的新 tag 只进入候选列表。
  • AI 调用配置可编辑；单条 memo 整理、标签总结 and 提醒抽取都支持单独选择大模型配置和提示词。
  • 按标签生成整体总结，可把同一标签下的零散 memos 整理成一条人可读的展示 memo。
  • 待澄清评论和 waiting_user 任务状态。
  • .txt、.md 附件解析到 artifacts 表；Word、Excel、PPT、PDF 附件通过 MinerU 服务转成 Markdown 参与整理；特别支持 .drawio（.drawio.svg）与 Mind Elixir .json 脑图本地高保真离线解析。
  • 管理端内置 AI 离线问答与 RAG Prompt 编译器，支持多标签药丸胶囊选择、模糊匹配、一键生成并复制完整 Prompt。

还没有完成：向量索引、自动文档、Memos 主库备份恢复。

1. 准备配置与各 Key 详细说明

复制环境变量示例并填写真实值。切勿把包含真实密钥的 .env 文件提交到 Git 仓库，确保数据隐私安全。

cp .env.example .env

下表为系统中涉及到的所有核心 Key 及环境变量的配置说明与使用指南：

变量/配置键使用类型详细说明与使用指南
SIDECAR_ADMIN_TOKEN 管理认证 Sidecar 专属管理接口和 /admin/ui 调试控制台的身份认证 Bearer 令牌。您可以将其设置为任意高强度的强随机密码字符串。该 token 由浏览器端读取并存入本地 localStorage 用于 API 交互鉴权，服务端绝不会主动泄漏此值。
MEMOS_BASE_URL 地址解析 Memos 服务的宿主机可访问地址。在 Docker 容器网络下，Compose 服务间已经自动将该值覆盖并连接为 http://memos:5230。如果您在非 Docker 环境下独立运行，请修改为对应的物理地址。
MEMOS_API_TOKEN 数据读写 Memos 用户的个人访问令牌（Personal Access Token / PAT）。用于 Sidecar 后台轮询或 webhook 状态下读取新 Memo 记录、将整理后的归档 Markdown 回写写入、建立 Memo 原生 REFERENCE 关联引用、发表澄清评论等核心读写操作。建议在 Memos 个人设置中生成永久有效的 PAT，避免使用可能随时失效的短期 Session 登录 Token。
MEMOS_WEBHOOK_URL 事件驱动 非必填。仅在 Sidecar 的 app.yaml 将内容摄入模式 memos.ingestion_mode 配置为 webhook 或 both 时有效。由于新版 Memos 默认会拦截 localhost 或 Docker 局域网私有 IP 的 Webhook 触发，如果您使用此模式，需要将此变量配置为公网可访问的 Sidecar 接收地址（或使用内网隧道反代服务生成的公网域名 URL）。在默认 poll 主动轮询模式下，此项可留空。
REMINDER_WEBHOOK_URL 定时推送 时间提醒（带有 #提醒 标签）功能的推送出口网关。符合 Bark 兼容接口规范。当任务后台检测到设定的闹钟/提醒时间到期后，系统会发起 POST 请求并自动拼接 title（任务标题）和 body（Memo 精炼正文）进行消息推送。常用的配置格式例如：https://api.day.app/your-bark-key/。
MINERU_API_TOKEN 文档解析 MinerU 文档高保真解析服务的 API Key。系统检测到上传了 PDF、Word、Excel 或 PPT 等富媒体附件后，会调用该服务进行在线精准转写排版，提取出格式工整的 Markdown 内容作为上下文供 AI 理解。您可以注册 MinerU 开发者平台并获取 API Key 填写至此。该 Key 会在运行期安全加载，不会保存进 app 配置文件或随备份包导出。
DEEPSEEK_API_KEY AI 智能整理 DeepSeek 大语言模型服务的 API Key。作为系统的默认 AI 整理引擎，默认使用极速、高性价比且通过 specific 机制优化了思维链（已在 extra_body 中禁用 thinking 模式以防止摘要阶段出现延迟和空响应错误）的 deepseek-v4-flash 官方模型。填入此 Key 即可瞬间拉起全自动的内容摘要、标签归纳与周总结。
/ OPENROUTER_API_KEY / OPENAI_API_KEY AI 备选模型 非必填。分别为中国移动 平台、OpenRouter 聚合模型入口以及 OpenAI 官方大模型的 API 密钥。可用作系统内置的备用 Provider，或者您可以在调试管理界面的“AI 配置”板块中为特定任务（例如只对提醒抽取、标签总结）单独绑定不同的第三方模型提供商。

2. Docker 容器化部署说明

Prism (棱镜) 采用先进 of Docker 容器拓扑进行标准化发布与本地沙箱运行。默认的部署栈包含 4 个高度隔离的容器服务：

📌 1. 容器部署拓扑架构

  • 网关服务 (gateway): 采用超轻量高性能的 Caddy 2 代理。作为系统唯一对外暴露的安全收敛入口，负责按同域路径进行安全隔离分流：/admin/*、/health、/webhooks/* 安全反向代理至 Sidecar 后台，根路径 / 转发至 Memos 容器。
  • 主应用 (memos): 提供核心笔记的在线发布、记录以及用户交互前端环境。
  • Sidecar 核心 API (sidecar): 宿主管理端，提供配置持久化、候选标签模块和离线 QA Prompt 编译器等轻量接口服务。
  • 后台工作流 (sidecar-worker): 异步任务消费协程，负责在空闲周期内轮询新 Memo、分析高保真图表大纲、向 LLM 请求摘要、解析提醒并触发 Webhook 推送。

🚀 2. Docker 部署常用 Nx 集成命令

项目所有的 Docker 容器操作均深度封装到了 Nx 任务管理器中，无需手动敲击繁琐的 compose 指令：

  • 一键启动/部署更新 (up): 
    npx nx run sidecar:up

    该命令将自动在后台拉起 4 个容器服务。默认通过网关安全收敛，只向宿主机暴露 http://localhost:8080 入口。Memos 默认的 5230 端口和 Sidecar 的 8080 内部端口均不暴露，完全由网关隔离保护。需要修改宿主机端口时，可指定环境变量运行：GATEWAY_PORT=9000 npx nx run sidecar:up。

  • 重新编译 Docker 镜像 (build):

    当您修改了底层源码或需要进行大版本发布更新时，可以通过 Nx 启动生产级全栈镜像打包构建：

    npx nx build sidecar
  • 检查部署健康状态: 
    curl http://localhost:8080/health

⚡ 3. 极速实机开发热重载配置 (P5 特性)

为了免除本地研发和快速验证时反复构建镜像（Build Image）的损耗，开发与测试环境中已对 docker-compose.yml 进行了极速优化：

  • 挂载本地源码：通过 - ./src:/app/src 卷挂载，本地 IDE 修改任何代码将即时反应在容器中。
  • Uvicorn 实时监听：Sidecar 容器启动命令已加载 --reload --reload-dir /app/src，一旦检测到本地代码修改，容器 Web服务将在 100 毫秒内轻量热重启并使代码生效，极大提高了实机部署的校验和研发效率。

🔒 4. Xget 镜像拉取加速保障 (Rule 5)

为了保障在国内或受限网络下依然能够实现容器镜像的秒级拉取，Dockerfile、docker-compose.yml 中的所有第三方公共镜像（如 Caddy、Memos、uv 运行底座等）均已默认配置了 Xget 加速分发源（如 xget.your-domain.com/cr/...），无需进行额外的 Docker Registry 配置即可畅享极速部署。

3. 打开调试管理页面

浏览器打开：

http://localhost:8080/admin/ui

页面用于调试 Sidecar：查看健康状态、查询任务、筛选任务状态、重试 failed 或 waiting_user 任务、查看候选标签并执行通过/拒绝、按标签生成整体总结，以及编辑 AI 调用配置。单条 memo 整理、标签总结和提醒抽取都可以在页面中单独选择大模型配置并编辑提示词。页面不会从服务器端读取 token；需要手动输入 SIDECAR_ADMIN_TOKEN，保存后只写入当前浏览器的 localStorage。

管理页面按「概览、任务、标签、AI 配置、模型、提醒、备份」聚合功能，并会根据浏览器宽度自适应布局：桌面宽度下列表和详情并排显示，窄屏下改为单列；任务、候选标签和提醒表格只在表格容器内横向滚动，不会撑开整个页面。

默认配置下，worker 空闲时会在 Memos 中自动创建或更新一条 #系统/Memosima 管理入口 memo（标题自动呈现为 Prism (棱镜) 管理入口），里面包含打开管理界面、查看任务、审核候选标签、生成标签总结和下载备份的链接。AI 整理 memo 底部也会附带管理链接；如果本次整理产生候选标签，会直接给出 /admin/ui#tag-candidates 审核入口。

管理页面支持锚点直达：#overview、#jobs、#tag-candidates、#tag-summary、#prompts、#models、#reminders 和 #backup。

4. 创建或更新 Memos PAT

可以用探针创建 PAT，并写到临时文件。命令输出默认不会显示 token 原文。

npx nx run sidecar:probe-memos -- \
  --bootstrap-username test \
  --bootstrap-password testtest \
  --create-pat \
  --pat-output-env-file /tmp/memosima-pat.env

把临时文件中的 MEMOS_API_TOKEN 写入本机受控的 .env 或部署密钥管理位置。

5. 配置内容入口

本地部署默认使用 poll 模式，Sidecar 会主动轮询 Memos 最近 memo，不需要公网 webhook，也不需要在 Memos Web 中配置内网回调地址。

如果要使用 Memos 内置 webhook，Memos 0.28.0 会拒绝 webhook URL 指向 localhost、Docker 内网或私有 IP，需要公网 URL，或者用公网隧道转发到 Caddy 网关公开入口。

npx nx run sidecar:probe-memos -- \
  --bootstrap-username test \
  --bootstrap-password testtest \
  --configure-webhook-url "$MEMOS_WEBHOOK_URL"

6. 处理一条 memo

在 Memos 中创建内容明确的 memo。默认轮询模式下，worker 会在短时间内发现新 memo，并创建 process_memo 任务；随后读取 memo、生成整理计划、创建 AI 整理 memo 并建立 relation。

查询任务：

curl -H "Authorization: Bearer<空格>$SIDECAR_ADMIN_TOKEN" \
  "http://localhost:8080/admin/jobs"

7. AI 自动生成 tag

配置默认 provider 对应的 API key 后，worker 会请求默认模型从 memo 正文中识别标签。AI 返回的 active_tags 只有已存在于正式标签集合时才会被直接使用；AI 发现的新标签会写入 candidate_tags 并进入审核列表，不会自动激活。后台会优先复用已有正式标签，并限制单次 AI 自动补充的标签数量，默认最多接收 5 个正式标签和 2 个候选标签。

默认配置 memos.show_candidate_tags: false 时，待审核业务标签不会以 #tag 形式写进 AI 整理 memo，因此不会在 Memos Web 中显示为可筛选标签；它们仍会在 Sidecar 管理页候选标签列表中等待审核。设为 true 后，待审核业务标签会像正式标签一样出现在 Memos Web 标签筛选中。

业务标签要求不同层级的最后一级名称唯一。例如已有正式标签 #项目/数管 时，后台自动整理遇到 #数管 或 AI 提出 #其他/数管，都会归一或拦截，不会再生成同名不同级别的新标签。

默认模型连接参数来自 config/models.yaml。当前默认 provider 为 DeepSeek deepseek-v4-flash，使用 https://api.deepseek.com 和 DEEPSEEK_API_KEY；DeepSeek 模板会通过 extra_body.thinking.type: disabled 关闭默认 thinking mode，避免单轮摘要和标签总结返回空内容或上游错误。同时保留 OpenRouter、、OpenAI 等 provider 模板。可以在管理页面“大模型”区域直接切换 provider、修改模型名、base URL、响应格式、扩展 JSON 和 key。保存后非机密参数写回 config/models.yaml，真实 key 写入本地 config/.env.local，不提交到 git，也不会进入备份包。

正式标签集合来自 config/taxonomy.yaml 中的 active 业务标签，以及已经通过审核写入 SQLite business_tags 表的标签。没有配置模型 key 时，系统仍会保留原有的显式 #tag 本地识别和模板整理。

AI 调用配置来自 config/prompts.yaml。在调试管理页面的“AI 调用配置”区域点击“保存 AI 配置”会写回该文件；每个调用点都可设置 provider 来单独选择 config/models.yaml 中的大模型配置，留空则使用默认 provider。点击任务“重试”时选择临时修改，只会把提示词写入当前任务 payload 的 llm_prompt_override，不会修改默认文件，也不会临时改变模型选择。单条 memo 整理可使用 {active_tags}、{local_plan_json}、{content} 占位符。

8. 按标签生成整体总结

打开 http://localhost:8080/admin/ui，在“标签总结”区域输入标签，例如 #项目/个人AI知识库 或高层级标签 #项目，设置最多读取 memo 数量后点击“生成总结”。Sidecar 会读取最近的 Memos 内容，在本地筛选出匹配该标签的 memo，调用默认模型生成 Markdown 总结，然后在 Memos 中创建一条新 memo。

标签总结支持层级匹配：输入 #项目 会包含 #项目 本身和 #项目/个人AI知识库、#项目/数管 等子标签下的普通 memo；#项目管理 这类相似但不是子路径的标签不会被纳入。如果原始 memo 没有手动标签，但它对应的 AI 整理 memo 命中了该标签，Sidecar 也会通过本地来源关系把原始 memo 纳入总结。

生成内容会带有 #系统/标签总结 和目标标签，标题形如 #项目/个人AI知识库 整体总结，正文包含模型整理后的总览、主题、进展、问题和相关 memo 列表。Sidecar 还会把总结 memo 通过 Memos 原生 REFERENCE relation 引用到每条参与整理的普通来源 memo，相关 memo 可在 Memos 界面中直接点击打开。标签总结不会把已有 #系统/AI整理 或 #系统/标签总结 memo 再次纳入来源；写入 relation 前也会跳过自引用、反向引用和多跳循环引用。标签总结使用 config/prompts.yaml 中的 tag_summary 配置，可以单独选择 provider，并可在管理页面保存提示词。模型上游请求失败时接口会返回 502 和固定错误信息，管理页会显示该错误，便于区分模型服务问题和 Sidecar 内部异常。

curl -X POST -H "Authorization: Bearer<空格>$SIDECAR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"tag":"#项目/个人AI知识库","limit":50}' \
  "http://localhost:8080/admin/tag-summaries"

9. 候选标签审核

memo 中出现未审核业务标签，或 AI 从正文提出新业务标签时，会进入候选标签列表。可以在 /admin/ui 中审核，也可以调用 API。

curl -H "Authorization: Bearer<空格>$SIDECAR_ADMIN_TOKEN" \
  "http://localhost:8080/admin/tag-candidates"

批准候选标签：

curl -X POST -H "Authorization: Bearer<空格>$SIDECAR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"note":"纳入正式标签"}' \
  "http://localhost:8080/admin/tag-candidates/1/approve"

批准后标签会写入 Sidecar 的 active 标签集合，后续 worker 会直接使用。

10. 待澄清任务

内容过短的 memo 会触发澄清评论，任务进入 waiting_user。问号、URL 参数等特殊字符不会单独触发待澄清。补充内容后可重试：

curl -X POST -H "Authorization: Bearer<空格>$SIDECAR_ADMIN_TOKEN" \
  "http://localhost:8080/admin/jobs/1/retry"

11. 附件解析

上传 .txt 或 .md 附件后，worker 会下载并解析为 Markdown artifact。上传 .doc、.docx、.xls、.xlsx、.ppt、.pptx 或 .pdf 后，worker 会先调用 MinerU 在线服务转成 Markdown。特别地，系统原生支持 .drawio（流程图，包括 .drawio.svg）与 Mind Elixir .json 思维脑图附件的 100% 本地离线高保真解析。后台会自动对 Draw.io 压缩流执行离线逆向 zlib 解压并提取 cell 组件文案，对脑图执行递归树形遍历，生成层级 Markdown 大纲写入 artifacts 数据库。解析后的内容会作为上下文一同拼装给 LLM 进行智能整理。

如果原始 memo 没有任何正文、只有附件，且附件解析和 LLM 分析成功，worker 会把 LLM 生成的标题以 Markdown 一级标题形式回写到原始 memo，例如 # 银行智能体白皮书。这样 Memos 列表中不再只显示空内容或附件名；完整整理结果仍会生成独立的 AI整理 memo，并通过原生引用关联回原 memo。

12. QA 离线问答 & Prompt 编译器

管理页面内置了专门为局域网或纯离线物理沙箱环境设计的 QA 离线问答面板（http://localhost:8080/admin/ui#qa）：

  • 多选药丸胶囊 UI：支持从下拉列表中选择 active 业务标签，或在输入框中直接输入任意模糊关键词（按回车转化为胶囊标签），且支持拼音与中文实时联想过滤。
  • 双路检索召回：后端会自动根据前缀 `#` 判断是精确标签匹配还是 Memos 正文/已解析附件大纲 Artifact 文本的模糊 contains 召回。
  • 一键生成与复制：自动编译出完美组装了「系统提示词」、「相关 Memo 正文与关联附件大纲上下文」以及「用户提问」的超级 Prompt，支持一键复制到剪切板，方便您直接粘贴到网页大模型中获取回答。

14. 多用户与安全隔离指南

⚠️ 隐私安全警示：
Prism 本质定位为个人 AI 知识库服务。在单 Sidecar 实例架构下，任务仅绑定单一 MEMOS_API_TOKEN。若多个用户在同一个 Memos 实例下直接共享该 Sidecar 实例，则由于系统标签治理缓存与解析后的附件大纲统一存放在同一个 SQLite 数据库中，在离线 QA 页面进行提问匹配时，可能会在上下文 Prompt 编译时召回其他用户的私密 Memo，导致严重的隐私泄露与数据混淆风险。目前系统不支持多账号无隔离直接混用。

🚀 官方推荐：独立容器化部署模式（SaaS/多租户物理隔离）

若您需要为多人、团队成员或不同账户提供各自独立的 AI 自动标签归档和离线 QA Prompter 功能，最安全、隔离性最好的方式是为每个用户部署一套物理隔离的独立容器栈：

  • 目录与代码物理隔离：为每个用户在宿主机创建独立的部署工作目录（例如 /home/abc/code/prism_userA），克隆相同的项目代码。
  • 独立配置文件：在各自的 .env 文件中填写该用户专属的 MEMOS_API_TOKEN、SIDECAR_ADMIN_TOKEN 以及各自偏爱的大模型 API Key。
  • 端口与反代隔离：在各自的 .env 中指定不同的 GATEWAY_PORT（例如用户 A 的网关暴露端口设为 8080，用户 B 设为 8081），在宿主机使用 Nginx 或反代工具将不同的二级域名（如 usera.memos.local）映射至对应的容器网关入口。
  • 数据防爆安全隔离：各实例的数据库、缓存文件、大模型 Key 物理隔离，100% 确保用户的隐私与数据所有权。
