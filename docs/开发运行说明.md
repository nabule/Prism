# 开发运行说明

## 当前阶段

当前实现范围为 P0+P1：

- FastAPI Sidecar API。
- Memos webhook 接收。
- SQLite 任务表、工作区表和 memo 映射表。
- 后台 worker 轮询处理 `process_memo` 任务。
- Memos API Client 基础封装。
- OpenRouter 模型配置预留。

AI 整理、标签审核、附件解析、RAG、自动文档和备份恢复属于后续阶段。

## 环境准备

本项目在 Linux 环境下开发和测试。所有常用开发命令通过 Nx 任务入口执行：

```bash
npx nx test sidecar
npx nx run sidecar:compile
npx nx build sidecar
```

不要直接运行 `pip install`；Python 依赖安装应封装在 Nx target、容器构建或后续统一脚本中。

## 启动 API

```bash
npx nx run sidecar:up
```

健康检查：

```bash
curl http://localhost:8080/health
```

## 启动 Worker

处理一条待处理任务：

```bash
npx nx run sidecar:probe-memos
```

持续轮询：

```bash
npx nx run sidecar:up
```

## 管理接口

查询任务：

```bash
curl -H "Authorization: Bearer $SIDECAR_ADMIN_TOKEN" \
  "http://localhost:8080/admin/jobs"
```

重试失败任务：

```bash
curl -X POST -H "Authorization: Bearer $SIDECAR_ADMIN_TOKEN" \
  "http://localhost:8080/admin/jobs/1/retry"
```

## 测试

```bash
npx nx test sidecar
```

当前已通过自动化测试：

```text
npx nx test sidecar
12 passed
```

## Docker 状态

仓库已提供 `docker-compose.yml` 和 `Dockerfile` 草案。Docker Hub 镜像下载优先使用 Xget 加速地址，例如 `xget.your-domain.com/cr/docker/neosmemo/memos:stable`。

当前真实 Memos 容器联调仍需重新执行 Compose 验收。最近一次 `docker compose pull memos` 与 `npx nx build sidecar` 已通过 `xget.your-domain.com` 成功完成镜像拉取和构建。
