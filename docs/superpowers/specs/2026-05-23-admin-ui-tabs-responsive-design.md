# 管理配置页标签页与响应式改造设计

## 背景

当前 `/admin/ui` 是单个长页，两栏网格中混排连接、健康状态、备份恢复、详情、大模型、AI 调用配置、任务、标签总结、候选标签和提醒等功能。页面可以完成调试和配置工作，但信息层级不够清晰，浏览器宽度变化时表格、按钮和表单容易拥挤。

本次改造采用「工作区式标签页」方案：保留现有单 HTML 管理页和后端 API，不改变管理接口语义；重点调整前端信息架构、响应式布局、锚点行为和视觉一致性。

## 目标

- 将管理能力按工作场景聚合到标签页，降低长页滚动和功能查找成本。
- 支持常见桌面、平板和手机宽度，避免正文、按钮、表格和详情区域互相挤压。
- 保留现有锚点直达能力，并修正当前锚点与卡片含义不一致的问题。
- 统一卡片、按钮、状态、表单、表格、提示和暗色模式的视觉样式。
- 不新增后端接口，不改变 token 存储策略，不引入前端构建链路。

## 非目标

- 不把 `/admin/ui` 拆成多页面应用。
- 不新增账号体系、服务端会话或 token 自动注入。
- 不重写管理 API。
- 不新增图表、统计聚合接口或实时推送。
- 不改动业务配置文件格式。

## 信息架构

页面分为固定顶部区域、标签导航和标签内容区。

固定顶部区域包含：

- 产品标题：`Memosima Admin`。
- 简短副标题：`Sidecar 管理配置`。
- Admin Token 输入、保存和清除操作。
- 全局通知区域，用于显示刷新、保存、错误等状态。

一级标签页：

| 标签页 | 包含功能 | 默认锚点 |
| --- | --- | --- |
| 概览 | 健康状态、最近任务入口、备份下载快捷入口 | `#overview` |
| 任务 | 任务筛选、任务列表、任务详情、重试 | `#jobs` |
| 标签 | 候选标签审核、标签总结生成、结果详情 | `#tag-candidates`、`#tag-summary` |
| AI 配置 | 单条 memo 整理、标签总结、提醒抽取的 provider 和提示词配置 | `#prompts` |
| 模型 | provider、Base URL、模型名、Key 状态、采样参数、扩展 JSON | `#models` |
| 提醒 | 提醒筛选、提醒列表、重试和取消 | `#reminders` |
| 备份 | 下载备份、上传恢复、恢复说明 | `#backup` |

详情展示保留为共享区域：

- 桌面宽度下，任务、标签、提醒和标签总结详情显示在右侧详情面板。
- 窄屏下，详情面板移动到当前标签内容下方。
- 详情内容继续使用格式化 JSON，不改变调试用途。

## 交互设计

标签页切换：

- 点击标签页时只切换当前页面内容，不刷新浏览器。
- URL hash 随标签页更新，便于从 Memos 管理入口直达。
- 页面加载时读取 hash 并打开对应标签页。
- 不识别的 hash 回退到 `#overview`。

锚点兼容：

- `#jobs` 打开「任务」标签。
- `#tag-candidates` 打开「标签」标签，并聚焦候选标签区域。
- `#tag-summary` 打开「标签」标签，并聚焦标签总结区域。
- `#models` 打开「模型」标签。
- `#backup` 打开「备份」标签。
- 新增 `#prompts` 和 `#reminders`。

数据加载：

- 页面打开后仍自动加载健康状态。
- 已保存 token 时，继续自动加载任务、候选标签、提醒、AI 配置和模型配置。
- 标签页切换不强制重新请求数据；用户点击刷新按钮时再刷新对应列表或配置。
- 保存模型配置后继续刷新健康状态，保持现有行为。

操作反馈：

- 全局通知保留，位置固定在顶部区域。
- 成功、错误、警告使用状态色和文字区分。
- 危险操作（恢复备份、取消提醒）继续要求浏览器确认。
- API Key 输入框仍只在用户填写时提交；留空不覆盖已保存 key。

## 响应式规则

桌面宽度（大于等于 1100 px）：

- 顶部区域横向排列标题、token 操作和全局通知。
- 标签内容采用 `minmax(0, 1fr)` 主内容和 360 px 详情侧栏。
- 表格允许横向滚动，但主布局不应撑破视口。

中等宽度（700 px 至 1099 px）：

- 顶部区域换行排列。
- 标签导航保持横向滚动。
- 内容使用单列布局，详情区域位于主内容之后。
- 表单工具栏允许换行，每个输入项保留合理最小宽度。

窄屏（小于 700 px）：

- 页面内边距减少。
- 标签导航横向滚动，按钮尺寸稳定，不因文字换行改变高度。
- 表格包裹在横向滚动容器中。
- 操作按钮换行排列，避免溢出。
- 长文本使用截断或换行策略，不覆盖其他内容。

## 视觉规范

- 保持安静、工具化的管理后台风格，不做营销式大标题或装饰图形。
- 卡片边框圆角不超过 8 px。
- 主色继续使用蓝色，但降低页面对单一蓝色的依赖，状态色用于成功、警告和错误。
- 表单、按钮、状态标签、表格和代码块使用统一间距。
- 暗色模式继续通过 `prefers-color-scheme: dark` 支持。
- 不引入图标库或外部资源，避免单文件管理页增加依赖。

## 实现边界

主要改动文件：

- `src/memosima/api/admin_ui.py`：重排 HTML 结构，调整 CSS，增加标签页切换逻辑，修正 hash 映射。
- `docs/使用手册.html`：更新管理页面说明，补充标签页和锚点说明。
- `docs/配置说明.html`：更新 `/admin/ui` 管理页面入口和锚点列表。

测试和验证：

- 通过 `npx nx test sidecar` 验证现有 API 与管理页返回不被破坏。
- 通过真实浏览器或 Playwright 打开 `/admin/ui`，检查桌面和移动宽度下的布局、标签切换和锚点直达。
- 验证 token 保存、健康状态刷新、任务刷新、候选标签刷新、提醒刷新、模型刷新、AI 配置刷新等现有入口仍可操作。

## 验收标准

- 打开 `/admin/ui` 默认显示「概览」标签页。
- 打开 `/admin/ui#jobs` 时直接显示「任务」标签页。
- 打开 `/admin/ui#tag-candidates` 和 `/admin/ui#tag-summary` 时直接显示「标签」标签页，并滚动到对应区域。
- 打开 `/admin/ui#models`、`/admin/ui#backup`、`/admin/ui#prompts`、`/admin/ui#reminders` 时显示对应标签页。
- 在 1366 px、900 px、390 px 宽度下，页面主体不出现非预期横向溢出；表格横向滚动限制在表格容器内。
- 任务、候选标签、提醒列表仍能展示详情，详情内容不挤压主列表。
- 保存 token、清除 token、刷新健康状态、刷新任务、刷新候选标签、刷新提醒、刷新模型、保存模型、刷新 AI 配置、保存 AI 配置、下载备份、上传恢复等按钮仍绑定正确行为。
- `npx nx test sidecar` 通过。
- 相关使用文档已同步更新。
