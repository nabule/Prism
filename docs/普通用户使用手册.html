Memosima 普通用户使用手册

普通用户日常只需要使用 Memos 客户端或 Memos Web 页面记录内容。Memosima Sidecar 会在后台自动读取新 memo，生成 AI 整理 memo、识别已有标签，并把新标签放入候选列表等待确认。

1. 日常使用入口

入口用途
Memos Web 或客户端日常记录、查看原始 memo、查看 AI 生成的整理 memo。这是普通用户的主要入口。
Memosima 管理页面 /admin/ui调试、查看后台任务、重试失败任务、审核 AI 提出的候选标签、按标签生成整体总结、备份和恢复 Sidecar 数据。普通记录不需要打开它。

管理员需要先部署好 Memosima、配置 Memos 访问 token 和模型 key。普通用户不需要接触 API key、Docker 或命令行。本地部署默认由 Sidecar 主动轮询 Memos，不需要公网 webhook。

2. 添加一条内容

在 Memos 中新建 memo，直接写正文即可。内容越清楚，AI 整理和标签识别越稳定。

今天调研了本地 AI 知识库方案。
Memos 负责快速记录，Sidecar 负责后台整理、生成摘要和标签建议。
下一步要验证附件解析和语义检索。

保存后，Sidecar 会在后台自动发现并处理这条 memo。处理完成后，Memos 中会出现一条新的 AI 整理 memo，并通过 Memos 原生 relation 关联到原始 memo。

3. 写内容的建议

  • 一条 memo 尽量只记录一个主题，例如一次会议、一段想法、一个资料摘录或一个任务进展。
  • 正文里写清楚背景、结论、下一步，AI 会更容易生成摘要、要点和待办。
  • 可以手动写已有标签，例如 #项目/个人AI知识库。已有正式标签会被直接使用。
  • 不确定用什么标签时可以不写，AI 会从正文中提出候选标签。
  • 短句或只有问题的 memo 可能会被标记为待澄清，需要补充信息后再重试。

4. AI 会生成什么

后台处理成功后，AI 整理 memo 通常包含这些内容：

  • 标题：根据正文生成的简短标题。
  • 摘要：对原文的结构化总结。
  • 要点：原文中的关键信息。
  • 待办：可以继续执行的事项。
  • 标签状态：已使用的正式标签、待审核的新标签、被禁用或无法使用的标签。
  • 原文摘录：便于回看来源。

如果模型 key 没有配置，系统仍会做本地模板整理和显式 #tag 识别，但不会调用大模型生成更完整的摘要和标签建议。

4.1 从 Memos 打开管理界面

管理员启用默认配置后，Memos 中会出现一条 #系统/Memosima 管理入口 memo，里面有打开管理界面、查看任务、审核候选标签、生成标签总结和下载备份的链接。AI 整理 memo 底部也会附带“打开管理界面”链接；如果本次整理产生候选标签，会额外显示“审核候选标签”链接。

这些链接会打开 /admin/ui，页面仍需要输入 SIDECAR_ADMIN_TOKEN。token 只保存在当前浏览器，不会写入 Memos memo。

5. 标签如何自动生成

Memosima 使用两层标签机制：

标签类型来源是否自动生效
正式标签管理员在配置中预置，或候选标签审核通过后形成。会直接使用
候选标签用户写了未审核的新 #tag，或 AI 从正文中提出新标签。不会直接生效，需要审核
禁用标签管理员明确禁用的标签。不会使用

只有原始用户 memo 没有任何业务标签时，AI 才会自动建议标签。AI 返回的 active_tags 只有命中正式标签集合时才会直接使用；AI 提出的新标签会进入 candidate_tags，不会自动变成正式标签。用户已经手写业务标签时，系统只使用用户标签，不再接收 AI 额外补充的标签。

AI 整理 memo 本身不会再写入业务 #tag，避免 AI 生成内容污染 Memos 标签筛选；它只保留 #系统/AI整理、#系统/标签待审核 等系统标签。业务标签会在“标签”区域以普通文本显示，管理员可以在 Memosima 管理页面审核候选标签。

同名业务标签不能同时出现在不同层级。比如系统已有 #项目/数管 时，用户只写 #数管，后台会归一到已有正式标签；AI 也不能再新增 #其他/数管 这类同名候选。

6. 手动写标签

可以在 Memos 正文中直接写标签：

完成 Memosima 管理页面调试。
#项目/个人AI知识库 #阶段/P2

如果 #项目/个人AI知识库 是正式标签，它会直接进入整理结果。如果 #阶段/P2 不是正式标签，它会进入候选标签列表，等待审核。

7. 不写标签时的自动识别

不写任何标签也可以。AI 会根据正文内容判断适合的正式标签，并在需要时提出新标签建议。

今天把 的 qwen 模型接入到知识库整理流程。
模型参数改成配置文件控制，并验证了真实接口调用。

如果系统已有 #项目/个人AI知识库，AI 可以直接使用它。如果 AI 认为需要 #模型/，但这个标签还不是正式标签，它会进入候选列表。默认每条无标签 memo 最多接收少量 AI 补充标签；用户手动写了业务标签时，AI 标签建议会被忽略。

8. 审核 AI 提出的新标签

有权限的用户可以从 #系统/Memosima 管理入口 memo 或 AI 整理 memo 底部链接打开管理页面，也可以直接访问：

http://localhost:8080/admin/ui#tag-candidates

在页面中输入 SIDECAR_ADMIN_TOKEN，进入“候选标签”区域。

  • 点击“通过”：候选标签会变成正式标签，后续同类 memo 可直接使用。
  • 点击“拒绝”：这次候选不会进入正式标签集合。
  • 审核备注可用于记录通过或拒绝的原因。

当前审核通过会写入 Sidecar 数据库，后续 worker 会使用；不会自动改写 config/taxonomy.yaml。

9. 待澄清内容怎么处理

如果 memo 太短、缺少上下文，或像是在问一个需要更多信息的问题，系统可能不会直接创建 AI 整理 memo，而是在原 memo 下创建澄清评论。

处理方式：

  1. 回到 Memos，编辑原始 memo，补充背景和目标。
  2. 打开管理页面 /admin/ui。
  3. 找到对应 waiting_user 任务，点击“重试”。

10. 附件内容

普通用户可以上传 .txt、.md、Word、Excel、PPT 或 PDF 附件。后台会下载附件并转成 Markdown artifact，其中 Office/PDF 使用 MinerU 在线服务解析。

解析后的 Markdown 会和 memo 正文一起进入 AI 整理流程，因此附件里的内容也会参与摘要、标签识别和 AI 整理 memo 生成。drawio 和 Mind Elixir 专用解析仍属于后续范围。

如果你只上传附件、不写正文，AI 分析成功后会自动给原始 memo 补一个标题，例如 # 项目方案。这个标题用于在 Memos 列表里快速识别附件内容；详细摘要、要点和标签仍在新生成的 AI整理 memo 中查看。

11. 使用提醒

如果管理员配置了提醒通知出口，可以在 memo 中写 #提醒 和明确时间：

#提醒 明天 09:30 提交周报
周报需要发给项目群，并同步本周风险。

Sidecar 会在后台按管理员配置的提醒抽取模型和提示词识别时间，并创建一次性提醒。到点后，系统会通过管理员配置的 webhook 发送通知，通知正文包含提醒内容、到期时间和来源 memo UID。

建议写清楚日期和时间，例如 #提醒 2026-05-22 14:00 跟进合同。如果只写“有空提醒我”或时间已经过去，系统会在原 memo 下评论要求补充具体时间，但普通 AI 整理仍会继续。

12. 常见问题

为什么没有生成 AI 整理 memo？

常见原因是 webhook 没有送达、worker 没有运行、模型 key 没配置、任务失败，或 memo 被判定为待澄清。可以在 /admin/ui 查看任务状态。

为什么新标签没有直接出现在正式标签里？

新标签必须审核。这样可以避免 AI 或用户临时写法污染正式标签体系。

我应该经常手动写标签吗？

已有明确标签时可以手动写；不确定时不用强行写，让 AI 先生成候选标签，再由管理员审核。

普通用户使用客户端还是 Web？

日常记录使用 Memos 客户端或 Memos Web 都可以。Memosima 的 Web 管理页面主要用于调试、任务重试和标签审核，不是日常记录入口。

13. 如何查看某个标签的整体总结

管理员可以在 /admin/ui 的“标签总结”区域输入一个标签，例如 #项目/个人AI知识库 或高层级标签 #项目，点击“生成总结”。系统会把该标签下最近的普通来源 memos 整理成一条新的 Memos memo；输入高层级标签时，也会包含其子标签下的 memo，例如 #项目/个人AI知识库。如果原始 memo 没有手动标签，但 AI 整理结果命中了该标签，也会纳入总结。总结 memo 会通过 Memos 原生引用关联到参与整理的 memo，普通用户可以在 Memos Web 或客户端里点击相关 memo 回看来源；已有 AI 整理和历史标签总结不会被再次作为来源。

14. 备份和恢复 Sidecar 数据

管理员可以在 /admin/ui 的“备份恢复”区域下载备份 ZIP。备份包含 Sidecar SQLite 数据和非机密配置文件，用于保留任务、候选标签、提醒、artifact 和 memo 映射等 Sidecar 状态。

恢复时选择备份 ZIP 并确认上传，系统只替换 Sidecar SQLite 数据库，不会自动覆盖配置文件，也不会备份或恢复 Memos 主库中的原始 memo。恢复前建议先下载一份当前备份。
