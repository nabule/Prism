Prism (棱镜)

🌈 Prism (棱镜)：光的折射与思维的绚烂
您随手丢进 Memos 的原始笔记，就像一束朴素的白光。当它穿过 Prism 时，被 AI 优雅地折射开来，化作了标签、结构化 Markdown（MinerU 解析）和清晰的知识脉络。零散的想法进去了，出来的是一抹绚丽的知识彩虹。

Prism 是基于 Memos 的个人 AI 知识库 Sidecar 服务。Memos 负责采集和保存原始知识，Sidecar (Prism) 负责 AI 整理、标签治理、澄清评论、附件解析和后续检索能力。

当前状态

已完成 P0、P1、P2 核心闭环，以及 P3 阶段附件离线高保真解析与离线 QA RAG 问答 Prompter。

  • Caddy 网关统一入口，默认从 http://localhost:8080/ 访问 Memos，从 /admin/*、/health、/webhooks/* 访问 Sidecar。
  • Memos webhook 接入和 SQLite 任务系统。
  • AI 整理 memo 创建，支持 DeepSeek、、OpenRouter 等 OpenAI-compatible LLM 和本地模板回退。
  • Memos 原生 REFERENCE relation 关联原始 memo 与 AI memo。
  • 内置 /admin/ui 调试管理页面，用于查看任务、重试任务、审核候选标签、备份和恢复 Sidecar 数据。
  • worker 可在 Memos 中自动维护 #系统/Memosima 管理入口 memo（标题自动呈现为 Prism (棱镜) 管理入口），AI 整理 memo 也会附带管理页面和候选标签审核链接。
  • 候选标签审核和审核后 active 生效。
  • 当用户原始 memo 没有业务标签时，LLM 可从正文自动建议正式标签和候选标签；新标签仍需人工审核。
  • LLM 提示词从 config/prompts.yaml 加载，管理页面可保存默认提示词，也可在重试时临时覆盖。
  • 待澄清评论和 waiting_user 状态。
  • 附件离线高保真解析： .txt、.md 附件解析为 artifact；Word/Excel/PPT/PDF 支持通过 MinerU 在线服务转换；特别原生支持 .drawio（包括 .drawio.svg）流程图解压提取与 Mind Elixir 思维脑图递归转换，100% 离线，零网络 Token 开销。
  • 离线 QA & Prompt 编译器： 提供专属的 RAG 问答页面，支持多标签药丸胶囊选择、拼音模糊过滤、双路精确/模糊检索召回，并一键编译与复制附带完整上下文的超级 Prompt。
  • #提醒 时间识别和一次性定时通知，支持 Bark 兼容 webhook。
  • 管理页面支持下载 Sidecar 备份包，并从备份包恢复 Sidecar SQLite 数据库。

未完成范围：向量索引、自动文档、Memos 主库备份恢复。

多用户与安全隔离说明

⚠️ 隐私安全警示：
Prism 本质定位为个人 AI 知识库服务。在单 Sidecar 实例架构下，任务仅绑定单一 MEMOS_API_TOKEN。若多个用户在同一个 Memos 实例下直接共享该 Sidecar 实例，则由于系统标签治理缓存与解析后的附件大纲统一存放在同一个 SQLite 数据库中，在离线 QA 页面进行提问匹配时，可能会在上下文 Prompt 编译时召回其他用户的私密 Memo，导致严重的隐私泄露与数据混淆风险。目前系统不支持多账号无隔离直接混用。

🚀 官方推荐：独立容器化部署模式（SaaS/多租户物理隔离）

若您需要为多人、团队成员或不同账户提供各自独立的 AI 自动标签归档和离线 QA Prompter 功能，最安全、隔离性最好的方式是为每个用户部署一套物理隔离的独立容器栈：

  • 目录与代码物理隔离：为每个用户在宿主机创建独立的部署工作目录（例如 /home/abc/code/prism_userA），克隆相同的项目代码。
  • 独立配置文件：在各自的 .env 文件中填写该用户专属的 MEMOS_API_TOKEN、SIDECAR_ADMIN_TOKEN 以及各自偏爱的大模型 API Key。
  • 端口与反代隔离：在各自的 .env 中指定不同的 GATEWAY_PORT（例如用户 A 的网关暴露端口设为 8080，用户 B 设为 8081），在宿主机使用 Nginx 或反代工具将不同的二级域名（如 usera.memos.local）映射至对应的容器网关入口。

快速开始

cp .env.example .env
npx nx run sidecar:up
curl http://localhost:8080/health

默认 Docker Compose 会启动 gateway、memos、sidecar 和 sidecar-worker。Caddy 网关只暴露一个宿主机入口：http://localhost:8080/ 进入 Memos，http://localhost:8080/admin/ui 进入 Sidecar 管理页面，http://localhost:8080/health 返回 Sidecar 健康状态。Memos 的 5230 和 Sidecar 容器内部 8080 默认不直接暴露到宿主机；需要改宿主机端口时可设置 GATEWAY_PORT。

调试管理页面不会读取服务器端密钥，需要手动输入 SIDECAR_ADMIN_TOKEN，并只保存在当前浏览器的 localStorage。页面也可以编辑默认 LLM 提示词，或在重试任务时只临时覆盖当前任务使用的提示词。默认开启 Memos 内管理入口，worker 空闲时会自动创建或更新一条 #系统/Memosima memo（标题呈现为 **Prism (棱镜)** 入口），入口链接由 app.public_base_url 生成。

npx nx test sidecar

提醒用法

在 memo 中写入 #提醒 和明确时间，例如 #提醒 明天 09:30 提交周报。worker 会在整理该 memo 时调用默认模型抽取时间，创建持久化提醒；到期后通过 REMINDER_WEBHOOK_URL 发送 title/body 表单通知。时间模糊或置信度不足时，Sidecar 会在原 memo 下评论要求补充，不会阻塞普通 AI 整理。

备份恢复

管理页面的“备份恢复”区域可以下载 Sidecar 备份 ZIP。备份包含 Sidecar SQLite 快照和非机密配置文件；恢复时只替换 Sidecar SQLite，不会自动覆盖配置文件，也不会备份或恢复 Memos 主库。
